vulnhub VM(XssAndMysql) writeup
September 5, 2016

After popping the SqlToShell box, I decided to practice a machine with a XSS vulnerability. I've never actually exploited a XSS vulnerability before, so I thought it would be nice to see what it can do. This machine proved to be a lot more difficult than the last one!

So after booting up the VM, I went to the IP in my local browser, and found another standard blogging website.

I found that the comment section was vulnerable to XSS, and fired up beef.

A couple seconds after I injected the script, a user popped up on beef! It seems like they're mocking visitors on the website.

I was able to steal the user's cookie and obtain access to the admin section of the website.

There was nowhere to upload files, but there was a sql injection vulnerability.

I used Sqlmap to try to upload a file to the current directory(which was found by inputting a single quote as the parameter) but it seemed like I didn't have write privileges.

I then used Burp's spidering tool and found some other directories. It turned out the CSS directory was writable.

I then uploaded a php shell into the CSS directory.



Although it was beginner level, this VM took quite a while and a bit of lateral thinking for me. Definitely would recommend!
