vulnhub VM(SqltoShell) writeup
July 8, 2016

I wanted to practice exploiting a website, and found a beginner's challenge on vulnhub. *cue clickbait title* You won't believe what happened next!

So after booting up the VM, I went to the IP in my local browser, and found a standard blogging website.

After clicking through the links and finding the vulnerable file(cat.php), I checked the tables and found the users table at the bottom.

I then obtained the password and username for admin access. The password is an md5 hash for "P4ssw0rd".

After getting access to the admin page, I found a place to upload files, and tried to upload a PHP shell. Unfortunately, they don't allow PHP files.

I tried changing the name for the file to a PHP3 file, and it was allowed.

Now, I had no idea where it was uploaded to. After looking at the source code for the previous images, I found that they were in /admin/uploads.

I now have a remote shell!

Highly recommend this VM. It was a great starting point to practice some SQL injection techniques I'd read about but never applied.
