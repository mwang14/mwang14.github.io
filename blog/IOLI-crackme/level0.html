Reverse Engineering with Radare2
December 25, 2016

Level 1(Stack0)

You can view the binary here.
To find the password, all we need to do is examine the file(crackme0x00-original) with the strings command, we can see that it contains the string "250382". By entering this string, we have found the correct password.
Next, we need to patch it so that the file accepts any password. By disassembling it with radare2, (after opening the file, the commands are "aa" and "pdf @ main" to see the main method), we can see how it checks the password.

| 0x0804845e c74424048f85. mov dword [esp + local_4h], str.250382
| 0x08048466 890424 mov dword [esp], eax
| 0x08048469 e8e2feffff call sym.imp.strcmp ; int strcmp(const char *s1, const char *s2);
| 0x0804846e 85c0 test eax, eax
| ,=< 0x08048470 740e je 0x8048480

Now, all we need to do is patch it. We can see that it simply compares the input with the hardcoded string "250382". If we can replace the je with a jmp, it should jump to the accepted portion every single time regardless of whether the string is equal to "250382"!
After opening the file in radare2, we can jump to the address of interest(0x08048470) using the command, "s 0x08048470". You should see that the address has now changed. We can see the next 15 bytes using the pD command. Once here, we can write the bytes "eb" using the command "wx eb"(write bytes) to change the je to jmp.
