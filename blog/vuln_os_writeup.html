vulnhub VM(vulnOS) writeup
September 21, 2016

I've had some down time since the start of school, so I decided to try out another VM. This was the first time I actually had to use exploit-db. Read on to find out about a vulnerability in OpenDocMan and a Linux privilege escalation exploit.

So after booting up the VM, I first started out by nmap'ing the box. There are three ports open: 22, 80, and 6667. I first tried going to the IP in my browser.

After poking around(the about page was really helpful), I found some hidden content in the documentation page. This gave me access to a new page and username info!

I went to the new page it gave me, which was MyDocMan 1.2.7.

I logged in as guest, and navigated to the document upload page. Hopefully, they won't sanitize the upload, and I'll be able to upload a PHP shell!

I tried to upload a PHP shell, but it was denied :'(.

After moping and poking around some more, I decided to search on exploit-db, and found that there was a SQL injection vulnerability in MyDocMan 1.2.7. You can check that vulnerability out here. I fired up SQLMap, and it worked!

After poking around the databases, I found a new user: webmin.

The md5 hash for the password was "webmin1980"

I logged in as webmin, and was able to change the file settings to allow myself to upload PHP files.

I then uploaded a PHP file!

Unfortunately, I couldn't find a way to run it...

I found an uploads section of the website, and found a bunch of .dat files. Apparently I wasn't going to be able to get a shell through file inclusion(or maybe it's possible, but beyond my current scope of knowledge). I haven't touched SSH yet though. I ssh'd into the machine with the webmin credentials next.

I found a root directory, but wasn't able to access it. My guess was that the flag was in that directory

If you look up the version of Linux on exploit-db(Linux Kernel 3.13.0 < 3.19), there is a local privilege escalation exploit with C. You can check that out here.

Because it's a C exploit, I checked if GCC was installed..

I copied the exploit into the /tmp directory, and compiled it..

I ran the exploit, and now had root access!

Finally, I was able to go to the root directory and read the flag.



This VM took quite a while, even though I didn't have to write any real exploitations D:. I am planning on getting my OSCP next summer, so hopefully this is all good practice!
